# Data Contract Template

Use this template to define data contracts for the SOLID.AI Data Spine.

For the complete YAML template, see:

**[→ Full Template on GitHub](https://github.com/gusafr/midora-solid-ai/blob/main/ADOPTION/TEMPLATES/data-contract-template.yaml)**

---

## Template Sections

The template includes:

1. **Contract Identity** - Name, version, type
2. **Semantic Definition** - Business meaning and relationships
3. **Schema** - Fields, types, constraints
4. **Ownership & Lifecycle** - Who owns, SLAs
5. **Quality Expectations** - Completeness, accuracy, freshness
6. **Privacy & Security** - Sensitivity, encryption, access
7. **Consuming Systems** - Who uses this data
8. **Observability** - Monitoring, alerts, lineage
9. **Evolution Strategy** - Versioning, deprecation
10. **Examples & Documentation** - Usage guides

---

## Usage Instructions

1. **Copy the template** to your data catalog
2. **Fill in each section** - use the [Data Contract Prompt](../prompt-templates/data-contract-design.md)
3. **Review with stakeholders** - producers, consumers, data team
4. **Publish to catalog**
5. **Implement validation** in code
6. **Version and update** as contract evolves
